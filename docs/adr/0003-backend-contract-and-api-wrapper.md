# 0003: Backend Contract and API Wrapper

## Status

Active.

## Decision

The backend OpenAPI contract is represented by generated
`src/apis/client.ts`. Local admin code imports DTO types from that file but does
not hand-edit it.

`lib/admin/api.ts` is the session-aware backend integration boundary for admin
workflows after login. The login action calls the backend email login endpoint
directly because no admin session exists yet.

## Implementation Shape

- `src/apis/client.ts` is generated by `yarn generate:api`.
- `lib/admin/api.ts` wraps backend endpoints with `fetch`, `cache: "no-store"`,
  JSON envelope parsing, bearer token attachment, and typed DTO return values.
- Backend 401 responses become `AdminSessionExpiredError`.
- Backend 403 responses become `AdminAccessDeniedError`.
- Other failed responses become `BackendApiError` with the response status.
- Token expiry helpers decode backend token payloads but do not verify JWT
  signatures locally; tokens must originate from trusted backend API responses.

## Current Admin Backend Calls

- Auth: `/v2/auth/email/login` from the login action, plus
  `/v2/auth/refresh` and `/v2/auth/logout` from `lib/admin/api.ts`
- Moderation: `/v2/admin/journeys/publish`,
  `/v2/admin/journeys/publish/:publicId`,
  `/v2/admin/journeys/publish/:publicId/review`
- Articles: `/v2/admin/articles`, `/v2/admin/articles/:articleId`

Public helper calls use `lib/public-api.ts` and public endpoints where needed
for admin-compatible data. They do not make this app the owner of public UX.

## Consequences

- Do not edit `src/apis/client.ts` by hand unless explicitly copying the
  current generated contract during a project split.
- Backend shape changes should update the backend contract first, regenerate the
  client, then adapt local wrappers and normalizers.
- Keep admin API error mapping explicit; avoid broad catches that hide auth,
  permission, validation, or not-found behavior.
- Contract, wrapper, or shared DTO consumer changes should run
  `yarn tsc --noEmit` plus the focused admin tests for the touched workflow.
